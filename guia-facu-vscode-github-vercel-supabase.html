0El mapa: cómo encaja cada pieza

Antes de tocar nada, entendé el circuito. Es más fácil cuando ves el todo:

TU PC (VS Code + Claude Code) │ escribís y probás el código localmente │ cuando algo funciona → git push ▼ GITHUB (guarda el código, todas las versiones) │ Vercel "escucha" el repo ▼ VERCEL (toma el código de GitHub y publica la web, te da una URL) │ la app, cuando necesita datos, le pega a → ▼ SUPABASE (la base de datos donde viven los datos)

La regla de oro: vos nunca subís nada "a mano" a Vercel. Trabajás en tu PC, hacés push a GitHub, y Vercel deploya solo. GitHub es la fuente de verdad del código; Supabase es la fuente de verdad de los datos.

Las tres cuentas (GitHub, Vercel, Supabase) tienen plan gratis que te alcanza de sobra. Truco: Vercel y Supabase te dejan registrarte con tu cuenta de GitHub, así que creás GitHub primero y las otras dos salen en un clic.

1Preparar la PC (instalar lo básico)

Vas a instalar 3 cosas: VS Code, Node.js y Git.

1.1 Visual Studio Code (el editor)

  1. Entrá a code.visualstudio.com
  2. Botón grande de descarga (detecta tu sistema: Windows / Mac).
  3. Ejecutá el instalador. En Windows, tildá estas opciones:
    • ✅ "Add to PATH"
    • ✅ "Add 'Open with Code' action to file context menu" (abrís carpetas con clic derecho)
  4. Abrí VS Code. Listo.

1.2 Node.js (para correr la app)

Casi todas las apps web modernas (React, Next.js, Vite) necesitan Node.

  1. Entrá a nodejs.org
  2. Descargá la versión LTS (la verde, estable — NO la "Current").
  3. Instalá con todas las opciones por defecto.
  4. Verificá: en VS Code abrí Terminal → New Terminal y escribí:
node --version
npm --version

Si te devuelve dos números de versión, está instalado.

1.3 Git (lo que conecta con GitHub)

  1. Windows: git-scm.com/download/win → instalá con opciones por defecto.
    Mac: en la terminal escribí git --version; si no lo tiene, te ofrece instalarlo solo.
  2. Verificá: git --version
  3. Configurá tu identidad (una sola vez):
git config --global user.name "Facu Rosello"
git config --global user.email "tu-mail@ejemplo.com"
Después de instalar Node y Git, cerrá y reabrí VS Code para que tome los cambios.

2Crear las cuentas

2.1 GitHub esta primero

  1. Entrá a github.com → Sign up.
  2. Mail, contraseña y un nombre de usuario prolijo (ej. facurosello).
  3. Verificá el mail.
  4. Anotá usuario y contraseña — la vas a usar para Vercel y Supabase también.

2.2 Vercel (el hosting)

  1. Entrá a vercel.com → Sign Up.
  2. Elegí "Continue with GitHub" → autorizá.
  3. Si te pregunta el tipo de cuenta, elegí Hobby (gratis).

2.3 Supabase (la base de datos)

  1. Entrá a supabase.com → Start your project.
  2. Elegí "Continue with GitHub" → autorizá.
  3. Ya estás adentro. (La base la creamos en el paso 7.)
En 5 minutos tenés las tres cuentas listas y conectadas entre sí.

3Bajar el proyecto de la nube a tu PC

Tu app hoy está en un entorno cloud (Lovable, Bolt, v0, Replit, donde sea). Casi todos tienen botón para exportar el código o conectar con GitHub directo. Dos caminos:

Camino A — El cloud ya conecta con GitHub el más fácil

Muchos (Lovable, Bolt, v0) tienen un botón "Export to GitHub" / "Connect GitHub":

  1. Apretalo, autorizá tu cuenta de GitHub.
  2. Te crea el repositorio solo con todo el código adentro.
  3. Saltá directo al paso 4.2 (clonar el repo).

Camino B — Descargar el código a mano (ZIP)

  1. Bajá el .zip del proyecto.
  2. Descomprimilo en una carpeta ordenada. Sugerencia: creá C:\Proyectos\ y te queda C:\Proyectos\mi-app\.
  3. Seguí con el paso 4.1.
Anotá qué tecnología usa tu app (React, Next.js, Vite…). Lo dice el archivo package.json. No hace falta entenderlo ahora, pero ayuda si algo falla.

4Abrir el proyecto en VS Code y correrlo local

4.1 Abrir la carpeta

  1. En VS Code: File → Open Folder → elegí la carpeta de tu app.
  2. Abrí la terminal: Terminal → New Terminal.

4.2 (Si usaste el Camino A) Clonar el repo

git clone https://github.com/TU-USUARIO/mi-app.git
cd mi-app

Después File → Open Folder sobre esa carpeta.

4.3 Instalar dependencias y correr

En la terminal, dentro de la carpeta del proyecto:

npm install

Esto baja todas las librerías que la app necesita (un par de minutos). Después:

npm run dev

Te muestra algo como Local: http://localhost:3000. Ctrl+clic en ese link → se abre tu app en el navegador, corriendo desde tu PC. 🎉

Si npm run dev da error, fijate qué dice el package.json en "scripts" — puede ser npm start en vez de npm run dev.

5Subir el código a GitHub

(Si usaste el Camino A, ya está en GitHub y te salteás esto — pero leé el flujo diario del final.)

5.1 Loguearte a GitHub desde VS Code

  1. Abajo a la izquierda, ícono de persona (Accounts) → Sign in with GitHub.
  2. Se abre el navegador → Authorize. VS Code quedó conectado.

5.2 Inicializar Git y subir

git init
git add .
git commit -m "Primera versión de la app"

Ahora subilo, lo más cómodo desde VS Code:

  1. Panel Source Control (ícono de ramas, o Ctrl+Shift+G).
  2. Botón "Publish to GitHub".
  3. Elegí "Publish to GitHub private repository" (privado = solo lo ves vos).
  4. En segundos tu código está en GitHub.
Antes de subir, asegurate de que exista un archivo .gitignore con node_modules y .env adentro. Eso evita subir basura y, sobre todo, tus claves secretas. Si no existe, creá uno con ese contenido.

6Conectar Vercel con GitHub (deploy automático)

Acá está la magia: Vercel toma tu repo y lo publica, y cada vez que subas cambios a GitHub, redeploya solo.

  1. Entrá a vercel.com (ya logueado con GitHub).
  2. Add New → Project.
  3. Te lista tus repos. Elegí el de tu app → Import.
  4. Vercel detecta solo qué framework es. No toques nada.
  5. Deploy.
  6. En 1-2 minutos te da una URL pública (tipo mi-app.vercel.app). Esa es tu app online. 🚀

A partir de ahora, cada git push dispara un deploy automático. No volvés a configurar nada.

Las variables de entorno (claves) se cargan en Vercel en Project → Settings → Environment Variables. Lo vemos en el paso 8, porque primero necesitás las de Supabase.

7Crear la base de datos en Supabase

  1. En supabase.com → New Project.
  2. Completá:
    • Name: nombre del proyecto (ej. mi-app).
    • Database Password: generá una fuerte (botón "Generate") y guardala.
    • Region: South America (São Paulo) sa-east-1 — mejor latencia desde Argentina.
  3. Create new project. Tarda ~2 minutos.

7.1 Crear las tablas

  • Visual: Table Editor → New table. Columnas con clics. Ideal para empezar.
  • SQL: SQL Editor. Pegás el SQL y lo corrés. Más rápido (y Claude Code te lo genera, paso 9).

7.2 Conseguir las credenciales

Settings (engranaje) → API. Anotá dos cosas:

  • Project URL (algo como https://xxxx.supabase.co)
  • anon public key (una clave larga)
La anon key es segura para el frontend. Hay otra, la service_role key, que es secreta total — NUNCA va en el código del navegador ni se sube a GitHub.

8Conectar la app con Supabase

8.1 Instalar la librería

npm install @supabase/supabase-js

8.2 Variables de entorno (las claves)

Creá un archivo .env (o .env.local en Next.js) en la raíz, con las credenciales del paso 7.2. El nombre depende del framework:

Vite / React:

VITE_SUPABASE_URL=https://xxxx.supabase.co
VITE_SUPABASE_ANON_KEY=tu-anon-key

Next.js:

NEXT_PUBLIC_SUPABASE_URL=https://xxxx.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=tu-anon-key
Verificá que .env esté en tu .gitignore. Las claves NUNCA se suben a GitHub.

8.3 Las mismas variables, en Vercel

Como .env no se sube, Vercel no las tiene. Cargalas a mano:

  1. Vercel → tu proyecto → Settings → Environment Variables.
  2. Agregá cada variable (mismo nombre y valor que en tu .env).
  3. Save. En el próximo deploy, la app online se conecta a Supabase.

8.4 Usarla en el código

import { createClient } from '@supabase/supabase-js'

const supabase = createClient(
  import.meta.env.VITE_SUPABASE_URL,      // o process.env.NEXT_PUBLIC_... en Next
  import.meta.env.VITE_SUPABASE_ANON_KEY
)

Esto no te lo tenés que pelear de memoria — Claude Code te lo arma (paso 9).

9Meter Claude Code dentro de VS Code

Claude Code es tu asistente IA dentro del editor: le pedís cosas en español y te escribe/corrige el código, crea las tablas, conecta Supabase, etc.

9.1 Instalar

  1. Panel Extensions (ícono de cuadraditos, o Ctrl+Shift+X).
  2. Buscá "Claude Code" (publicado por Anthropic).
  3. Install.
  4. Aparece el ícono de Claude en la barra lateral → Sign in → logueate con tu cuenta de Anthropic. Si no tenés, te la crea en el momento.
Alternativa por terminal (opcional): npm install -g @anthropic-ai/claude-code y el comando claude. Pero con la extensión alcanza.

9.2 Cómo se usa

Abrís el panel de Claude Code y le hablás normal:

  • "Conectá esta app a Supabase usando las variables del .env"
  • "Creá una tabla de usuarios con nombre, email y fecha de alta, dame el SQL"
  • "¿Por qué me da error este componente?" (le pegás el error)
  • "Subí los cambios a GitHub"

Ve tus archivos, propone los cambios, y vos aprobás. Como tener un programador al lado.

9.3 (Opcional, recomendado) Conectar el MCP de Supabase

Un MCP es un "enchufe" que le da a Claude Code acceso directo a una herramienta. El de Supabase le permite ver y manejar tu base de datos solo, sin copiar y pegar SQL.

  1. Conseguí un access token en Supabase: Account → Access Tokens → Generate new token. Copialo.
  2. En la raíz del proyecto creá .mcp.json (reemplazá el token):
{
  "mcpServers": {
    "supabase": {
      "command": "npx",
      "args": ["-y", "@supabase/mcp-server-supabase@latest"],
      "env": {
        "SUPABASE_ACCESS_TOKEN": "tu-token-aca"
      }
    }
  }
}

Reiniciá VS Code. Ahora Claude Code consulta y modifica tus tablas directo.

Agregá .mcp.json al .gitignore también — tiene un token adentro.

10El flujo de trabajo diario

Una vez montado todo, tu día a día es este loop simple:

  1. Programás/probás en tu PC con npm run dev (lo ves en localhost).
  2. Cuando algo te gusta, guardás los cambios en GitHub. Desde Source Control:
    • Escribís un mensaje corto (ej. "agregué login").
    • Commit → Sync/Push.
    • (O le decís a Claude Code "subí los cambios a GitHub".)
  3. Vercel deploya solo en ~1 minuto → tu URL pública se actualiza.
  4. Los datos viven en Supabase, los ves y editás en su panel.

Eso es todo. Local → GitHub → Vercel, una y otra vez. La base la tocás solo cuando cambiás la estructura de datos.

Checklist rápido de instalación

  • VS Code instalado
  • Node.js (LTS) instalado — node --version responde
  • Git instalado y configurado (nombre + mail)
  • Cuenta GitHub creada
  • Cuenta Vercel (login con GitHub)
  • Cuenta Supabase (login con GitHub)
  • Proyecto bajado del cloud y abierto en VS Code
  • npm install + npm run dev corriendo en local
  • Código subido a GitHub (repo privado)
  • Proyecto importado en Vercel → URL pública andando
  • Base de datos creada en Supabase (región São Paulo)
  • .env con las claves de Supabase (y en .gitignore)
  • Mismas variables cargadas en Vercel
  • Extensión Claude Code instalada y logueada
  • (Opcional) .mcp.json con el MCP de Supabase

Reglas de seguridad (no te las saltees)

  1. .env y .mcp.json SIEMPRE en .gitignore. Nunca subas claves a GitHub.
  2. La service_role key de Supabase es secreta total — solo en variables de servidor, jamás en el navegador.
  3. Si por error subiste una clave a GitHub, rotala (generá una nueva) — borrarla del repo no alcanza, ya quedó en el historial.
  4. Repos en privado salvo que quieras que sea open source.
Cualquier traba, el orden de ataque es: leé el error en la terminal → pegáselo a Claude Code → si es de la base, mirá el panel de Supabase → si es de deploy, mirá los logs en Vercel (Deployments → el deploy fallido → ver logs).
